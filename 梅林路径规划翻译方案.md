# 梅林路径规划 → Segment 翻译方案

外部规划器 `mf_action_planner` 发送原始动作序列，BT 引擎消费结构化 segment JSON。本文档定义翻译规则。

---

## 一、输入格式

**Topic**: `/mf_action_seq` | **类型**: `std_msgs/Float32MultiArray`

每条动作 8 个元素：`[arg0, arg1, arg2, arg3, arg4, arg5, arg6, arg7]`

### move (arg0=0)

| 参数 | 含义 |
|------|------|
| arg1 | 格子行 row (0~5) |
| arg2 | 格子列 col (0~2) |
| arg3 | 目标高度 (mm: 0/200/400/600) |
| arg4 | 朝向 (弧度) |

### fetch (arg0=1)

| 参数 | 含义 |
|------|------|
| arg1 | 格子行 row |
| arg2 | 格子列 col |
| arg3 | 相对高度差 (目标-当下, mm) |
| arg4 | 朝向 (弧度) |

### 地图坐标系

**蓝区**：
```
row=5  | (5,2,000) (5,1,000) (5,0,000) | 出口区
row=4  | (4,2,200) (4,1,400) (4,0,200) |
row=3  | (3,2,400) (3,1,600) (3,0,400) | 梅林区
row=2  | (2,2,200) (2,1,400) (2,0,600) |
row=1  | (1,2,400) (1,1,200) (1,0,400) |
row=0  | (0,2,000) (0,1,000) (0,0,000) | 入口区
```

**红区差异**：row=2 为 `(2,2,600) (2,1,400) (2,0,200)`

---

## 二、输出格式

**Topic**: `/planning/segments` | **类型**: `std_msgs/String` (JSON)

```json
{
  "stage": "MEILIN",
  "plan_id": "meilin_001",
  "segments": [...]
}
```

### 梅林区 segment_type

| segment_type | 用途 | 必填字段 |
|--------------|------|----------|
| MOVE2 | 同层移动 | move_target |
| CLIMB | 上下台阶 | move_target, climb_mode, climb_direction, climb_height |
| GRASP | 抓取 KFS | height_diff (可选) |
| STORE1 | 存入车体 | wait_result (可选) |
| STORE2 | 暂存手臂 | wait_result (可选) |

---

## 三、坐标计算规则

### 3.1 格子尺寸

- 每个格子：**1.2m × 1.2m**
- KFS 位置：格子中心
- 世界坐标原点：由定位系统定义

### 3.2 格子中心坐标计算

设 `(0,0)` 格子中心为 `(x0, y0)`，则格子 `(row, col)` 的中心坐标：

```
x = x0 + row × 1.2
y = y0 + col × 1.2
```

### 3.3 抓取位置计算

**关键约束**：
- 车身必须**正对** KFS（yaw 指向格子中心）
- 车停在**靠近 KFS 的格子边线**外一定距离

**可配置参数**：`grasp_distance` — 车身距离格子边线的距离（默认 0.4m）

**计算方法**：

设 KFS 位于格子 `(row, col)`，车身朝向 yaw：

1. KFS 格子中心：`(x_kfs, y_kfs)`
2. 根据朝向确定边线位置（车身从哪个方向接近）
3. 车身位置 = 边线位置 + grasp_distance

**朝向与边线对应关系**：

| yaw 范围 | 接近方向 | 边线位置 | 车身坐标计算 |
|---------|---------|---------|-------------|
| ≈0 (朝+X) | 从下方接近 | y = y_kfs - 0.6 | x = x_kfs, y = y_kfs - 0.6 - grasp_distance |
| ≈π (朝-X) | 从上方接近 | y = y_kfs + 0.6 | x = x_kfs, y = y_kfs + 0.6 + grasp_distance |
| ≈π/2 (朝+Y) | 从左侧接近 | x = x_kfs - 0.6 | x = x_kfs - 0.6 - grasp_distance, y = y_kfs |
| ≈-π/2 (朝-Y) | 从右侧接近 | x = x_kfs + 0.6 | x = x_kfs + 0.6 + grasp_distance, y = y_kfs |

> 格子边线到中心距离 = 0.5 × 1.2m = 0.6m

**示例**：抓取 (1,1) 格子的 KFS，yaw=0（从下方接近）

```
格子中心: (2.4, 2.4)
边线位置: y = 2.4 - 0.6 = 1.8
车身位置: x = 2.4, y = 1.8 - 0.4 = 1.4
车身朝向: 0 (正对 KFS)
```

### 3.4 上下台阶位置计算

**关键约束**：
- 车身应在台阶**边线中间**上下，确保稳定

**台阶边线位置**：
- row=i 和 row=i+1 之间的台阶边线：`x = x0 + (i+0.5) × 1.2`
- Y 方向取格子列中心：`y = y0 + col × 1.2`

**CLIMB 的 move_target**：
- 上台阶：目标格子的入口边线中间
- 下台阶：目标格子的出口边线中间

**示例**：从 (0,0) 上台阶到 (1,0)

```
台阶边线: x = x0 + 0.5 × 1.2 = 1.8  // 假设 x0=1.2
CLIMB 起点位置: (1.8, 1.2)  // col=0 的中心 y
CLIMB 终点位置: 进入 (1,0) 后的位置
```

---

## 四、翻译规则

### 4.1 move (arg0=0) → MOVE2 或 CLIMB

**步骤**：
1. 根据目标 `(row, col)` 计算格子中心坐标
2. 比较高度变化，判断 MOVE2 或 CLIMB
3. CLIMB 需计算台阶边线中间位置

| 高度变化 | 生成 segment | climb_height |
|---------|-------------|--------------|
| 无变化 | MOVE2 | - |
| 增加 | CLIMB (UP) | 目标高度 - 当前高度 |
| 减少 | CLIMB (DOWN) | 当前高度 - 目标高度 |

### 4.2 fetch (arg0=1) → GRASP + STORE

**步骤**：
1. 计算 KFS 格子中心坐标 `(x_kfs, y_kfs)`
2. 根据 `grasp_distance` 和 yaw 计算车身停靠位置
3. 生成 MOVE2 先移动到抓取位（如果需要）
4. 生成 GRASP（携带 height_diff）
5. 生成 STORE（根据策略选择 STORE1/STORE2）

**注意**：
- fetch 前必须有对应的 move 到正确位置和朝向
- 翻译器可检查位置是否匹配，不匹配则插入补充 MOVE2

---

## 五、配置参数

```yaml
# 梅林区翻译配置
meilin:
  # 格子参数
  grid_size: 1.2              # 格子尺寸 (m)
  grid_origin: {x: 1.2, y: 1.2}  # (0,0) 格子中心坐标

  # 抓取参数
  grasp_distance: 0.4         # 车身距离格子边线的距离 (m)

  # 存储参数
  max_body_capacity: 3        # 车体最大存储数
```

## 六、完整示例

### 示例1：简单抓取

**输入**：
```
动作1: [0, 1, 1, 0, 0.0]     → 移动到 (1,1) 高度0
动作2: [1, 1, 1, 0, 0.0]     → fetch (同一格子，高度差0)
```

**计算**：
- 格子尺寸 1.2m，原点 (1.2, 1.2)
- KFS 中心：x = 1.2 + 1×1.2 = 2.4, y = 2.4
- yaw=0（从下方接近），边线：y = 2.4 - 0.6 = 1.8
- grasp_distance = 0.4m，车身位置：(2.4, 1.8 - 0.4) = (2.4, 1.4)

**输出**：
```json
{
  "stage": "MEILIN",
  "plan_id": "meilin_001",
  "segments": [
    {"segment_type": "MOVE2", "move_target": {"x": 2.4, "y": 1.4, "yaw": 0.0}},
    {"segment_type": "GRASP", "height_diff": 0},
    {"segment_type": "STORE1", "wait_result": false}
  ]
}
```

### 示例2：跨层移动

**输入**：
```
动作1: [0, 1, 1, 0, 0.0]     → 移动到 (1,1) 高度0
动作2: [0, 2, 1, 200, 0.0]   → 上台阶到 (2,1) 高度200
动作3: [1, 2, 1, 0, 0.0]     → fetch
```

**计算**：
- (1,1) → (2,1)：高度变化 0→200，需 CLIMB
- 台阶边线：x = 1.2 + 1.5×1.2 = 3.0，y = 1.2 + 1×1.2 = 2.4

**输出**：
```json
{
  "segments": [
    {"segment_type": "MOVE2", "move_target": {"x": 3.6, "y": 1.4, "yaw": 0.0}},
    {"segment_type": "CLIMB", "move_target": {"x": 3.0, "y": 2.4, "yaw": 0.0},
     "climb_mode": "UP", "climb_direction": "FORWARD", "climb_height": 200},
    {"segment_type": "MOVE2", "move_target": {"x": 3.6, "y": 1.4, "yaw": 0.0}},
    {"segment_type": "GRASP", "height_diff": 0},
    {"segment_type": "STORE1", "wait_result": false}
  ]
}
```

### 示例3：带高度差的抓取

**输入**：
```
动作1: [0, 2, 1, 200, 1.57]  → 移动到 (2,1) 高度200
动作2: [1, 2, 1, -100, 1.57] → fetch 下降抓取
```

**计算**：
- KFS 中心：x = 1.2 + 2×1.2 = 3.6, y = 2.4
- yaw=1.57≈π/2（从左侧接近），边线：x = 3.6 - 0.6 = 3.0
- grasp_distance = 0.4，车身位置：(3.0 - 0.4, 2.4) = (2.6, 2.4)

**输出**：
```json
{
  "segments": [
    {"segment_type": "CLIMB", "move_target": {"x": 3.0, "y": 2.4, "yaw": 1.57},
     "climb_mode": "UP", "climb_height": 200},
    {"segment_type": "MOVE2", "move_target": {"x": 2.6, "y": 2.4, "yaw": 1.57}},
    {"segment_type": "GRASP", "height_diff": -100},
    {"segment_type": "STORE1", "wait_result": false}
  ]
}
```

## 七、标定数据

```yaml
# 梅林区配置
meilin:
  grid_size: 1.2              # 格子尺寸 (m)
  grid_origin: {x: 1.2, y: 1.2}  # (0,0) 格子中心坐标
  grasp_distance: 0.4         # 抓取时车身距离格子边线 (m)
  max_body_capacity: 3        # 车体最大存储数
```

> **说明**：格子尺寸 1.2m，坐标可由 `grid_origin + row/col × grid_size` 计算，无需逐格标定。
